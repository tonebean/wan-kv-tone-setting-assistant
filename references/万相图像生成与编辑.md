# 万相图像生成与编辑

这份文件不是官方长文镜像，只保留这套 skill 当前真正依赖的调用事实。

## 上游官方文档

这套 skill 当前默认参考的 Wan 2.7 图像模型文档入口：

- `https://waytoagi.feishu.cn/wiki/CNjhwJdNBiyiGpkuODMcn8R0nNd`

如果这份短说明和官方文档发生冲突，优先以官方文档为准。

## 当前默认使用范围

这个 skill 当前默认只依赖：
- Wan2.7 文生图
- 异步任务查询
- 结果图及时保存

首图之后，如果路线已经对了但还差局部完成度，也可以进入：
- 图像指令编辑
- 图像交互式编辑

图像编辑、组图、多图参考都不是这套 skill 的默认主链路；要扩展时再查官方完整文档。

## 上游文档里明确提到的能力范围

按这份 Wan 2.7 image 发布文档，模型本身是图像生成与编辑的统一模型，包含：

- 标准版
- PRO 版
- 文生图
- 文生组图
- 图生组图
- 图像指令编辑
- 图像交互式编辑

文档还明确提到：

- 标准版支持 `1K / 2K`
- PRO 版支持 `4K`
- 自带全通道透明底
- 强调极速秒级出图

这套 skill 当前只取其中最适合品牌 KV 主线的部分：

- `wan2.7-image-pro`
- 文生图
- 异步任务
- 结果及时保存

## 当前默认模型

```text
wan2.7-image-pro
```

## 当前默认地址

北京：

```text
https://dashscope.aliyuncs.com/api/v1
```

新加坡：

```text
https://dashscope-intl.aliyuncs.com/api/v1
```

不要混用地域的 Key 和地址。

## 当前默认调用链路

### 1. 创建任务

```text
POST /services/aigc/image-generation/generation
```

当前 skill 默认按异步方式调用。

### 2. 查询任务

```text
GET /tasks/{task_id}
```

## 当前最重要的请求事实

- 请求头需要带 `Authorization: Bearer $DASHSCOPE_API_KEY`
- 文生图主内容放在 `input.messages`
- 这套 skill 当前默认最小参数为：
  - `size: 2K`
  - `n: 1`
  - `watermark: false`
  - `thinking_mode: true`

## 当前最重要的响应事实

- 创建成功后先保存 `task_id`
- 任务完成后，图片链接在：

```text
output.choices[0].message.content[0].image
```

- 结果链接有时效，默认按 24 小时理解
- 图片要尽快下载，不要只留临时 URL

## 当前 skill 对结果的处理原则

1. 先拿结果图
2. 立即落盘保存
3. 再判断是否符合场景
4. 先判断是局部细节问题，还是方向性问题
5. 路线对了才做局部编辑；路线错了先回头改需求梳理和执行描述
6. 不机械连续重跑
