# 如何获取 API Key

这不是官方长文摘录，而是给 agent 和调用者的最短可执行说明。

## 什么时候需要读这份文件

只要当前机器没有可用的 `DASHSCOPE_API_KEY`，就先停在这里，不继续生成。

## Agent 应该怎么引导用户

如果用户要继续测试这个 skill，但本机没有现成密钥，直接这样引导：

1. 打开阿里云百炼 API Key 页面：
   `https://bailian.console.aliyun.com/cn-beijing?spm=a2c4g.11186623.0.0.60905ec6PYDomF&tab=model#/api-key`
2. 选择准备实际调用的地域
3. 进入 API Key 页面
4. 创建一个可用于该地域的 API Key
5. 复制后不要发到长期文档或仓库里
6. 如果只是当前测试，直接把 Key 发给 agent，继续当前会话
7. 如果之后还要长期本地使用，再看 `API Key配置到环境变量.md`

如果 agent 支持直接展示链接，应该优先把上面的地址直接发给用户点击跳转，而不是只说“去控制台找”。

## 必须提醒用户的三件事

1. 北京和新加坡等不同地域的 Key 不能混用  
   Key 属于哪个地域，调用时就必须用该地域对应的地址。

2. 不要把真实 Key 写进 skill 文件  
   开源、文档分享、截图分享时都只保留占位符，不暴露真实值。

3. 如果是团队账号，先确认自己有创建 API Key 的权限  
   没权限时，不要在 agent 里硬猜，直接让用户去控制台确认账号权限。

## 这个 skill 默认怎么处理没密钥的情况

默认不继续假装执行，不编结果，不伪造调用。

这是硬规则：

- 没有可用 Key，就停止真实调用
- 先引导用户完成获取
- 测试时优先让用户把 Key 直接发到当前会话
- 长期使用时再引导配置环境变量
- 配好之前，不输出任何“已经生成成功”的口吻

而是明确告诉用户：
- 你当前还没配置可用密钥
- 请先打开上面的 API Key 页面获取密钥
- 如果只是当前测试，拿到后直接发给我
- 如果要长期本地使用，再配置环境变量
- 有了可用 Key 之后再继续测试
